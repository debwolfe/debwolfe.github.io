Like a fish needs a bicycle basket

Enumerable's Cycle Method

December 15, 2014

Okay. Okay. Perhaps the Cycle Method is a bit more useful to a Ruby programmer than a bicycle is to a fish. But there are so many other methods in the Enumerable class to know and like. In IRB entering: 

 Enumerable.instance_methods.sort
Returns this array: 

          [:all?, :any?, :chunk, :collect, :collect_concat, :count, :cycle, :detect, :drop, :drop_while, :each_cons,
           :each_entry, :each_slice, :each_with_index, :each_with_object, :entries, :find, :find_all, :find_index,
            :first, :flat_map, :grep, :group_by, :include?, :inject, :lazy, :map, :max, :max_by, :member?,
            :min, :min_by, :minmax, :minmax_by, :none?, :one?, :partition, :reduce, :reject,
            :reverse_each, :select, :slice_before, :sort, :sort_by, :take, :take_while, :to_a, :zip]

There's map for when you want to derive something new based on every element in your enum basket. All the new items, after you've run a block on them, will return in a new array. There's also group_by for when you want to lump your enum basket into piles based on an attribute of the items. After you run the condition block, you will get back a hash for each kind of item. If all goes well, the condition is the key to the hash and each of the items will be stored in the value part as an array.

The Cycle Enumerable is a bit different in that we just want to hop through the elements in the enum basket for a specified number of hops and run a block. How do we know if our basket is a special Enumerable basket? It will respond to .each. But in the case of cycle we can give it a separate number of times to cycle through the collection. When nill is passed, the cycle will run on forever and ever in a bad way. Cycle seems to be good for text and display tasks. The TextHelper in Rails also has a Cycle method. If we take our cohort list, let's cycle through everyone and print them a message:
